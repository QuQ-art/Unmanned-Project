# 无人系统设计 - 第05次作业实现说明

## 项目概述

本项目实现了基于强化学习（PPO/SAC/TD3算法）的战机对抗控制系统，目标是训练智能体控制战机击落远处的固定靶机。

## 实现的核心功能

### 1. 初始化函数 (`utils/initialize.py`)

**功能**：生成战机和靶机的初始状态

**实现要点**：
- 我方战机起始位置：原点附近 (0, 0, 50)，高度500m
- 敌方靶机位置：距离我方120-150单位（1200m-1500m），满足≥1000m的要求
- 随机化敌机位置的水平角度和垂直角度，增加训练多样性
- 我方战机初始速度：20单位/s，朝向+x方向
- 敌方靶机：缓慢移动或悬停，速度较低

**关键代码逻辑**：
```python
# 距离随机化
distance = np.random.uniform(120, 150)  # 1200m-1500m
angle_horizontal = np.random.uniform(-np.pi/3, np.pi/3)
angle_vertical = np.random.uniform(-np.pi/6, np.pi/6)
```

---

### 2. 观测处理函数 (`utils/observation.py`)

**功能**：将原始状态转换为智能体的观测空间（15维）

**观测设计**（15维向量）：
1. **相对位置** (3维)：敌机相对我方的xyz坐标，归一化到[-1,1]
2. **相对距离** (1维)：使用tanh归一化
3. **朝向角度** (1维)：我方朝向与目标方向的cos值，范围[-1,1]
4. **我方姿态角** (3维)：roll, pitch, yaw，归一化到[-1,1]
5. **我方速度** (4维)：速度量级 + 三轴速度分量u,v,w
6. **我方角速度** (2维)：ω和β
7. **敌方血量** (1维)：归一化到[0,1]

**设计思路**：
- 使用相对位置而非绝对位置，提高泛化能力
- 计算朝向角度的cos值，直接反映瞄准精度
- 所有观测值归一化到[-1,1]范围，便于神经网络训练

---

### 3. 奖励函数 (`utils/reward.py`)

**功能**：计算每步的奖励，引导智能体学习正确的行为

**奖励组成**（共8个分量）：

| 分量 | 说明 | 权重/范围 |
|------|------|-----------|
| distance | 距离改善奖励（势能函数） | 系数0.5 |
| proximity | 距离接近奖励（指数衰减） | 0-2.0 |
| angle | 朝向角度奖励 | 0-2.0 |
| damage | 击中敌机奖励 | 伤害×0.1 |
| kill | 击毁敌机奖励 | 100.0 |
| time_penalty | 每步时间惩罚 | -0.1 |
| self_damage | 自身受伤惩罚 | 伤害×-0.05 |
| death | 自身坠毁惩罚 | -50.0 |
| speed | 速度奖励/惩罚 | -0.5 to 0.2 |
| altitude | 高度惩罚 | -1.0 to 0.0 |
| attitude | 姿态惩罚 | -0.5 to 0.0 |

**设计思路**（参考作业提示）：
1. **距离奖励**：势能函数形式，距离减少给予奖励
2. **角度奖励**：鼓励战机朝向敌机
3. **血量奖励**：击中敌机给予大奖励，击毁给予巨大奖励
4. **回合惩罚**：每步-0.1，鼓励快速完成任务
5. **安全约束**：惩罚过低高度、过度机动等危险行为

---

### 4. 终止条件函数 (`utils/truncate.py`)

**功能**：检查是否需要提前终止episode

**终止条件**：
- 高度过低 (< 50m) 或过高 (> 3000m)
- 距离敌机过远 (> 5000m)
- 姿态失控（roll > 162°或pitch > 108°）
- 速度异常（< 1或> 100）
- 飞出边界（|x|或|y| > 5000m）

**设计目的**：
- 防止智能体学习到危险或无效的行为
- 加快训练速度，避免无意义的长episode

---

### 5. 动作处理函数 (`utils/action.py`)

**功能**：将智能体输出的动作转换为平台控制指令

**动作映射**：
- **油门**：从[-1,1]映射到[0.3, 1.0]，避免油门过低导致失速
- **俯仰/滚转/偏航**：保持[-1,1]范围，添加clip限制

**关键设计**：
```python
# 油门下限0.3，避免失速
throttle = 0.3 + 0.35 * (action[0] + 1.0)
```

---

### 6. 训练主程序 (`main.py`)

**算法选择**：PPO（Proximal Policy Optimization）
- 也提供了SAC和TD3的配置（注释掉）

**网络结构**：
- Actor网络：[256, 256, 128] - 3层全连接
- Critic网络：[256, 256, 128] - 3层全连接
- 激活函数：Tanh

**训练参数**：
- 学习率：3e-4
- 批次大小：512
- 更新步数：2048
- Epoch数：10
- 折扣因子γ：0.99
- 熵系数：0.01（鼓励探索）

**回调函数**：
1. `CheckpointCallback`：每50000步保存模型
2. `RewardComponentsCallback`：记录奖励分量到CSV

**训练步数**：1,000,000步（可根据需要调整）

---

### 7. 测试脚本 (`test.py`)

**功能**：
- 加载训练好的模型
- 进行多个episode的测试
- 统计成功率、平均奖励等指标
- 实时显示奖励分量

**使用方法**：
```bash

```

## 设计思路总结

### 观测空间设计
- 使用**相对观测**而非绝对观测，提高泛化能力
- 包含**位置、速度、姿态、目标信息**，信息充分
- 所有观测**归一化**到[-1,1]，便于神经网络学习

### 奖励函数设计
- **多目标平衡**：接近目标 + 瞄准精度 + 击中奖励
- **势能函数**：距离减少持续给予奖励
- **稀疏奖励密集化**：击中给予大奖励，但也有持续的距离/角度奖励
- **安全约束**：惩罚危险行为（过低高度、失速等）

### 动作空间设计
- 4维连续控制：油门、俯仰、滚转、偏航
- **油门下限保护**：避免失速
- 控制量平滑映射，避免剧烈变化

### 网络结构选择
- **3层深度网络** [256, 256, 128]
- 比浅层网络有更强的表达能力
- 适合复杂的空战对抗任务

## 训练建议

### 训练流程
1. 在客户端创建房间，获取端口号
2. 修改`config/envs.yaml`中的端口号
3. 运行`python main.py`开始训练
4. 使用tensorboard监控训练：`tensorboard --logdir=./logs`
5. 训练完成后运行`python test.py`测试模型

### 参数调优建议
1. **如果训练不稳定**：降低学习率到1e-4
2. **如果探索不足**：增加熵系数ent_coef
3. **如果收敛太慢**：增加批次大小或减少n_epochs
4. **如果奖励波动大**：调整奖励函数的权重

### 分阶段训练策略（可选）
1. **阶段1**：简化任务，固定初始位置，只训练转向能力
2. **阶段2**：增加距离，训练接近能力
3. **阶段3**：完整训练，随机初始化位置
